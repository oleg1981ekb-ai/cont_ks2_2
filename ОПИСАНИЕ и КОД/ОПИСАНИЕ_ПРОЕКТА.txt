1. ОБЩАЯ СУТЬ И СТЕК ТЕХНОЛОГИЙ
Проект представляет собой автоматизированную систему контроля строительной исполнительной документации, актов и справок выполнения (КС-2, КС-3). 
- Язык разработки: Python 3
- Ключевые библиотеки: openpyxl (генерация Excel)
- База данных: Локальный структурированный файл JSON
- Развертывание/Бэкап: Автоматическая отправка в репозиторий GitHub
- Среда просмотра отчетов: LibreOffice Calc / MS Excel (macOS)

2. АРХИТЕКТУРА ДАННЫХ (Single Source of Truth)
Проект полностью переведен на принцип "Единого источника правды". Вся информация физически изолирована от Excel-отчетов и хранится в иерархическом дереве файла `database.json`.
Структура дерева (матрешка):
  Направление (например, "01_АНАПА (Таманская)")
    └── Подобъект (например, "01_271_КН")
          └── Месяц (например, "Февраль")
                ├── sum (float: бюджет периода, поддерживает копейки)
                └── status (dict: {"value": 3, "date": "01.07"})

3. СТРУКТУРА И ГРУППИРОВКА СТРОК В EXCEL (5 УРОВНЕЙ OUTLINE)
Итоговый Excel-отчет генерируется с нуля по данным из JSON. Для удобства свертывания блоков используются встроенные уровни структуры (Outline Level) от 0 до 4:
- Столбец A: Сквозная нумерация. Номер присваивается ТОЛЬКО строкам Месяцев, остальные строки остаются чистыми для визуального порядка.
- Столбец B: Наименование элементов структуры:
  ├── Уровень 0: Направление (Синяя заливка)
  │     └── Уровень 1: Подобъект (Светло-серая заливка)
  │           └── Уровень 2: Месяц (Пастельная заливка, вывод суммы с пробелами-разделителями)
  │                 └── Уровень 3: Документы (Чек-лист: Акт КС-2, Справка КС-3, Счет, Счет-фактура, ИД)
  │                       └── Уровень 4: Логгер (Серая строка курсивом: "└─ Дата изменения СтрК: 01.07")

4. МАСКИ ПОВЕДЕНИЯ (ЦИФРОВЫЕ ТРАФАРЕТЫ)
Чтобы исключить человеческий фактор и хаос при заполнении статусов, в `config.py` зашиты маски доступов для колонок документов (от СтрК до Оплаты). 
Если для конкретного документа колонка заблокирована регламентом (например, Оплата для Исполнительной Документации), генератор:
- Принудительно очищает ячейку от текста.
- Окрашивает её в сплошной пастельно-серый цвет блокировки (`#EAEAEA`).
Разрешенные ячейки остаются белыми и принимают чистые числовые статусы (1, 2, 3), на которые моментально срабатывает условное форматирование цвета LibreOffice Calc.

5. МОДУЛЬНАЯ СТРУКТУРА ПРОЕКТА
Код жестко декомпозирован на компактные независимые файлы по принципу разделения ответственности:
- `wizard_manager.py` — Главный пульт управления. Легкий скрипт, содержащий только чистый цикл главного меню (Пункты 1-5).
- `wizard_menus.py` — Интерфейс подменю. Управляет шагами добавления веток и изменения данных. Переведен на полностью цифровой кнопочный выбор (выбор месяцев по номерам 1-12, выбор объектов по спискам) для защиты от опечаток.
- `db_core.py` — Техническое ядро базы данных. Отвечает за загрузку/сохранение JSON, валидацию копеек, хронологическую сортировку месяцев по календарю года и расчет дат.
- `builder.py` — Чистый сборщик структуры Excel. Отвечает за обход JSON дерева, фиксацию верхней строки шапки таблицы (`freeze_panes`) и создание динамического колонтитула с датой выгрузки.
- `excel_styler.py` — Модуль стилей. Изолирует правила шрифтов, заливок и логику парсинга статусов для ячеек.
- `git_deployer.py` — Модуль автоматического деплоя на GitHub с тегированием версий.
- `config.py` — Глобальные константы, палитры цветов, списки документов и заголовки шапки.

6. ПОДПРАВЛЕННЫЕ БАГИ И СТАБИЛИЗАЦИЯ НА MAC
В ходе разработки были успешно решены специфические проблемы macOS и openpyxl:
- Исправлен американский формат вывода денег (запятые заменены на русские пробелы-разделители).
- Налажен ввод десятичных копеек как через точку, так и через русскую запятую без падения скрипта.
- Устранена критическая ошибка импорта `TextBlock` и `append` в классах `RichText` путем вынесения логов дат на отдельный 5-й уровень структуры в виде чистых текстовых строк.
- Добавлен файл `.gitignore`, который блокирует засорение репозитория GitHub бинарными файлами `.xlsx` и системным кэшем `__pycache__`.
